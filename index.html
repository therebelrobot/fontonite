Fontonite
A public API to convert TTF and OTF fonts to Web Fonts.

Why

I've found working with designers and front-end developers that there is a knowledge gap, specifically when it comes to web fonts. Designers worry about them only so far as to pick a font and deliver it to the developers, while the developers normally lack the tools and know-how to convert whatever esoteric filetype the font originally came in to the proper cross-browser formats.

CDNs (Google Web Fonts, TypeKit, etc.) have in part solved this issue in providing a simple mix-in to cover all cases. But what if the design calls for one not provided by a CDN? There are a number of OSS tools to assist with this process, but finding and installing all these toold can be an unnecessary hassle.

Fontonite aims to fill that gap.

Fontonite is an easy to use public API that aims to convert any font into web-safe formats (ttf, svg, woff, and eot) It utilizes publicly available OSS tools to convert the file to the needed formats and provides a .zip file with not only those files, but a sample css file to include in your stylesheet.

Other services may provide similar solutions, but from my experience, they place unneeded and undesired license scraping, which makes it difficult to work with a font when you actually have obtained legal rights to use it on the web. By using this tool, you automatically agree that you have legally obtained rights to copy/convert the font, so I have removed any restriction along those lines.

Alternatively, if you would like to run the API internally, the server is based on Node.js/io.js and is ISC licensed, so it is super simple to install locally and use.

Finally, I will be working to expand the versitility of this tool, to allow for more types of input fonts, as well as adding additional features like multi-font uploading, css customization, and glyph filtering. If there is a feature you would like to see in Fontonite, feel free to open a Github Issue, or, better yet, open a pull request!
